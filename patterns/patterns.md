# Паттерны проектирования

Реализации различных паттернов на Go.

## Что реализовано

### Concurrency (Конкурентность)

Паттерны для работы с горутинами и каналами:

1. **Fan-In** — объединение нескольких каналов в один
2. **Fan-Out** — распределение задач между воркерами
3. **Worker Pool** — пул воркеров для параллельной обработки
4. **Semaphore** — ограничение количества параллельных операций
